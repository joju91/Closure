# Manifest — Principles & Guidelines

Core governing principles for product, UX, psychology, ethics, and operational decision‑making.

---

## 🧭 Foundation

| Category | Rule / Principle |
|---------|------------------|
| Purpose | Help people manage practical tasks after a death |
| Scope | Practical only — not therapy |
| Goal | Minimize mental load for the user |
| Vision | Automated handling (1‑click in the future) |
| Tone | Warm, clear, trustworthy |

---

## 🧠 User Psychology

| Category | Rule / Principle |
|---------|------------------|
| Primary state | Overwhelmed (high cognitive load) |
| Secondary state | Unsure about correct order |
| Tertiary state | Afraid of missing something important |
| Core need | Avoid thinking — the system leads |
| Goal | The user should feel in control |

---

## 🔧 Decision Principles

| Category | Rule / Principle |
|---------|------------------|
| Decision | Anything that does not reduce mental load → remove it |
| Decision | The system determines the next step — the user does not choose |
| Decision | Action > information |
| Decision | Simplicity > correctness |

---

## 🤖 Automation Levels

| Category | Rule / Principle |
|---------|------------------|
| Current | Assisted (prepare, suggest) |
| Fallback | Guided (step‑by‑step instructions) |
| Future | Agent (fully automated) |

---

## 💬 Copy & Tone

| Category | Rule / Principle |
|---------|------------------|
| Copy | “Do this now” |
| Copy | “The next step is…” |
| Copy | No fluff — no unnecessary words |
| Copy | Short and clear |
| Copy | Decision‑driven language |
